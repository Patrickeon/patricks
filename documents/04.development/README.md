# 04.development/

> 개발 산출물(AN·DS·DV·TS·IM)을 **흐름 순서**로 둔다. 폴더 접두 번호가 곧 수행 흐름이다.
> Forward·Internal은 풀세트, Reverse는 역방향 중심, Operation은 참조용.

| 하위 | 무엇 |
|---|---|
| `01.분석/` | 요구사항 수집·정의·명세·추적 (AN) |
| `02.설계/` | IA·아키텍처·DB·API·화면·연동 설계 (DS) |
| `03.구현/` | 환경 구축·백엔드·프론트·퍼블리싱·코드리뷰 (DV) |
| `04.테스트/` | 시험 계획·단위·통합·성능·결함·릴리스·보안 (TS) |
| `05.이행/` | 마이그레이션·배포·UAT·운영 전환·매뉴얼 (IM) |
