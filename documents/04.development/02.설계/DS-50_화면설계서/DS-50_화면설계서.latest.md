## 1. 공통 설계 원칙

### 1.1 전역 레이아웃 구조

```
AppLayout.vue (Tauri Window, min: 1280×768)
├── AppHeader.vue          — 워크스페이스 진입 후 고정 표시
│   ├── 브랜드 마크 (A 아이콘 + 프로젝트명)
│   ├── 상태 뱃지 (n/7 READY 카운터, 부팅 완료 후)
│   └── 네비게이션 버튼 (산출물 | 레드마인 | 설정 | 팀 종료)
├── <router-view>          — 화면별 컴포넌트
└── AppToastContainer.vue  — 전역 토스트 (우측 하단)
```

### 1.2 색상 시스템

| 토큰 | 값 | 용도 |
|------|----|----|
| `--bg-base` | `#0f1117` | 앱 배경 |
| `--bg-panel` | `#1a1d27` | 패널 배경 |
| `--border` | `rgba(255,255,255,0.08)` | 패널 테두리 |
| `--text-primary` | `#e8eaf0` | 기본 텍스트 |
| `--text-muted` | `#6b7280` | 보조 텍스트 |
| `--accent` | `#6366f1` (Indigo) | 주요 버튼·강조 |
| `--ok` | `#22c55e` | READY 뱃지 |
| `--busy` | `#fbbf24` | BUSY 뱃지 |
| `--error` | `#ef4444` | ERROR 뱃지 |

### 1.3 공통 컴포넌트 규격

| 컴포넌트 | 규격 |
|---------|------|
| `StatusBadge` | `🟢 READY` / `🟡 BUSY` / `🔴 ERROR` — 원형 도트 + 텍스트 |
| `Button.primary` | Indigo 배경, 흰 텍스트, border-radius 6px |
| `Button.danger` | Red 배경, 흰 텍스트 |
| `Button.ghost` | 투명 배경, 테두리 있음 |
| `Toast` | 우측 하단 슬라이드인, 3초 자동 소멸 (error는 수동 닫기) |
| `Modal` | 중앙 오버레이, ESC 닫기 가능 |
| `Input` | `--bg-base` 배경, `--border` 테두리, focus 시 Indigo 테두리 |

│  └──────────────────────────────────┘    │

### 2.2 컴포넌트 명세

| 컴포넌트 | 타입 | 바인딩 | 비고 |
|---------|------|--------|------|
| 앱 로고 | 정적 | — | `A` 마크 + "AgiTeamBuilder Desktop" |
| 버전 표시 | 정적 | `appVersion` (Tauri API) | `v{major}.{minor}.{patch}` |
| RecentProjectList | 리스트 | `projectStore.recentProjects[]` | 최대 10개, 내림차순 |

| 새 프로젝트 버튼 | Button.primary | `@click → /settings` | — |
| 기존 열기 버튼 | Button.ghost | `@click → Tauri dialog.open()` | `.json` 필터 |

### 2.3 이벤트 처리

| 이벤트 | 처리 |
|--------|------|
| 최근 프로젝트 행 클릭 | `projectStore.load(path)` → 성공 시 `/boot` 이동, 실패 시 Toast(error) |
| 새 프로젝트 버튼 | `router.push('/settings?mode=new')` |
| 기존 열기 버튼 | `dialog.open({filters:[{name:'agiteam.json',extensions:['json']}]})` → 선택 시 `/settings?path=<path>` |
| 경로 유효하지 않음 | Toast(error, "프로젝트 경로를 찾을 수 없습니다. 제거하시겠습니까?") + 목록에서 제거 옵션 |

## 3. Screen-02: 프로젝트 설정

**라우트**: `/settings`  
**컴포넌트**: `SettingsView.vue`

### 3.1 레이아웃

```
┌────────────────────────────────────────────────┐
│ [← 뒤로]    프로젝트 설정                        │
├──────────┬─────────────────────────────────────┤
│ 탭 메뉴  │                                      │
│ 기본정보  │  [탭 선택에 따른 폼 섹션]              │
│ PM 설정  │                                      │
│ 팀 구성  │                                      │
│ 타이밍   │                                      │
│ 페르소나 │                                      │
├──────────┴─────────────────────────────────────┤
│              [저장]      [팀 부팅 시작 →]        │
└────────────────────────────────────────────────┘
```

### 3.2 탭별 필드 명세

#### 탭 1: 기본 정보 (`project.*`, `persona.*`)

| 필드 | 컴포넌트 | JSON 키 | 검증 |
|------|---------|---------|------|
| 프로젝트 이름 (내부) | Input | `project.name` | 필수, 영문+숫자+하이픈 |
| 프로젝트 표시 이름 | Input | `project.displayName` | 필수 |
| 워크스페이스 이름 | Input | `project.workspace.name` | 필수 |
| 워크스페이스 색상 | ColorPicker | `project.workspace.color` | Indigo/Red/Green/Blue 선택 |
| 페르소나 디렉토리 | PathInput + 폴더 선택 | `persona.dir` | 필수, 존재하는 경로 |
| Shared 공통 파일 | PathInput + 파일 선택 | `persona.commonFile` | 필수, .md 파일 |

#### 탭 2: PM 설정 (`pm.*`)

| 필드 | 컴포넌트 | JSON 키 | 검증 |
|------|---------|---------|------|
| PM 이름 | Input | `pm.name` | 필수 |
| 에이전트 타입 | Select | `pm.agent` | claude/codex/gemini |
| 실행 명령어 | Input | `pm.command` | 필수 |
| 시작 메시지 | Textarea (4행) | `pm.startupMessage` | 선택 |
| 시작 파일 목록 | DraggableFileList | `pm.startupFiles[]` | 각 경로 .md 파일 |

#### 탭 3: 팀 구성 (`team[]`)

각 역할 카드:

| 필드 | 컴포넌트 | JSON 키 | 검증 |
|------|---------|---------|------|
| 역할 ID | Input | `team[].role` | 필수, 영문 |
| 담당자 이름 | Input | `team[].name` | 필수 |
| 에이전트 타입 | Select | `team[].agent` | claude/codex/gemini |
| 실행 명령어 | Input (접힘) | `team[].command` | 필수 |
| 레이아웃 슬롯 | Select | `team[].layout` | middle_top/mid/bottom, right_top/mid/bottom |

- 역할 추가: `+ 역할 추가` 버튼 → 빈 카드 하단 추가
- 역할 삭제: 카드 우측 `×` → `"정말 삭제하시겠습니까?"` 확인 Modal

#### 탭 4: 타이밍 (`settings.*`)

| 필드 | 컴포넌트 | JSON 키 | 기본값 | 범위 |
|------|---------|---------|--------|------|
| READY 대기 (초) | NumberInput | `settings.readyTimeout` | 30 | 10~300 |
| 부팅 후 지연 (초) | NumberInput | `settings.postLaunchDelay` | 3 | 0~30 |
| READY 신호 최대 (초) | NumberInput | `settings.readySignalTimeout` | 60 | 10~600 |
| 자동 Enter 최대 | NumberInput | `settings.maxAutoSubmits` | 5 | 0~20 |

### 3.3 이벤트 처리

| 이벤트 | 처리 |
|--------|------|
| 저장 버튼 | 검증 → `invoke('save_config', {path, config})` → Toast(ok, "저장됨") |
| 팀 부팅 시작 | 저장 후 `/boot` 이동 |
| 페르소나 파일 경로 클릭 | 사이드 패널에서 해당 파일 열기 (Screen-05 연동) |
| 검증 실패 | 해당 필드 하단 인라인 에러 메시지 |

## 7. Screen-04b: 팀원 채팅 패널 (×6)

**컴포넌트**: `RoleChatPanel.vue`

### 7.1 레이아웃 (각 패널, 균등 크기)

```
┌────────────────────────────┐
│ DeveloperBE · 박개발  [🟢] │ ← PanelHeader (역할 + 담당자 + 에이전트 + 상태)
├────────────────────────────┤
│  [대화 로그 / 스트리밍]      │ ← ChatLog
│  PM: 환경 설정 완료했나요?   │
│  박개발: 네, DV-10 완료했…  │
├────────────────────────────┤
│ [메시지 입력]  [전송]  [⤢]  │ ← Input + 최대화 버튼
└────────────────────────────┘
```

### 7.2 상태 뱃지 전이 규칙

| 에이전트 출력 패턴 | 상태 전이 |
|-----------------|----------|
| `READY: {role}` | → 🟢 READY |
| 스트리밍 시작 | → 🟡 BUSY |
| 스트리밍 종료 | → 🟢 READY |
| 오류 패턴 (`Error:`, `Fatal:`) | → 🔴 ERROR |
| 승인 프롬프트 (`[Y/n]`, `Continue?`, `Allow`) | → Toast 알림 + 자동/수동 Enter 선택 |

### 7.3 패널 최대화

- 최대화 버튼(⤢) 클릭 → `workspaceStore.maximizedRole = role`

## 11. 브라우저 (독립 이동 창) — #17 A안 (2026-07-11 현행화)

**사이드바 패널(컨트롤)**: `BrowserPanel.vue` · **렌더 창**: 독립 최상위 `WebviewWindow`

> **설계 전환(#17)**: 기존 "사이드바에 핀 고정된 내장 WebView" 모델을 폐기하고, 브라우저를 **타이틀바를 가진 독립 최상위 창**으로 분리했다. 유저가 창을 OS 네이티브 드래그로 화면 어디로든 자유 이동·리사이즈한다. 앱은 창 위치/크기를 강제하지 않는다(따라이동·`browser_resize` 로직 전면 제거). Tauri IPC 계약은 DS-60 §3.8, API는 DS-40 §9 참조.

### 11.1 레이아웃

```
[사이드바 "브라우저" 패널]              [독립 브라우저 창 (자유 이동)]
┌───────────────────────────────┐   ┌─ ○ ○ ○ ──────────────────────┐ ← 타이틀바(OS 드래그)
│ [← ][→ ]  [https://...     ]  │   │                              │
│  주소/네비 컨트롤 바            │   │   Tauri WebviewWindow        │
├───────────────────────────────┤   │   (URL 렌더 영역)            │
│  상태: "별도 창에서 열려 있음"  │   │                              │
│  (또는 홈 화면 / 로딩 표시)     │   │   decorations:true           │
└───────────────────────────────┘   │   resizable:true · parent 없음│
  ※ 탭 전환해도 독립 창은 유지        └──────────────────────────────┘
    (isOpen은 workspaceStore 보관)     ※ 닫기: 창 타이틀바 닫기 버튼
```

### 11.2 컴포넌트 명세

| 컴포넌트 | 기능 |
|---------|------|
| AddressBar | URL 입력 + Enter 이동(`browser_navigate`) + 뒤로(`browser_back`)/앞으로(`browser_forward`) |
| 상태 표시 | 브라우저 열림 시 "별도 창에서 열려 있습니다" 안내, 미열림 시 홈/로딩 표시 |
| 독립 창(WebviewWindow) | `browser_open`으로 생성(초기 960×720, 이후 위치/크기 사용자 소유)·표시, `browser_close`로 종료. `browser:navigation` 이벤트로 주소창 동기화 |

> **상태 관리**: 브라우저 열림 여부(`isOpen`)는 `workspace` 스토어(Pinia)에 보관하여, 사이드바 탭을 전환해 패널이 언마운트/재마운트돼도 실제 독립 창 상태와 UI가 일치한다. 탭 전환만으로 창을 닫지 않는다. 닫기 경로는 ① 패널 주소창의 × 버튼(`closeBrowser`, status-hint에 "× 버튼으로 닫으세요" 안내) ② 독립 창 타이틀바 닫기 버튼 ③ 앱(메인 창) 종료 시 동반 종료 — 3가지다.
>
> **폐기 항목**: 기존 `ContextSendMenu`(텍스트 선택 컨텍스트 전송)·`🔄새로고침` 버튼·핀 고정 인라인 WebView는 현행 `BrowserPanel.vue`에 존재하지 않는다(#17 전환으로 제거).
