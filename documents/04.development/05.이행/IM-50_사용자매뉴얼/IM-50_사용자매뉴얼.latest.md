## 1. 앱 개요

AgiTeamBuilder Desktop은 AI 에이전트 팀(PM + 6역할)을 GUI 환경에서 운영하는 크로스플랫폼 데스크톱 앱입니다. 기존 bash+cmux 기반 CLI 시스템을 Windows와 macOS 모두에서 동일하게 동작하는 설치형 앱으로 재설계했습니다.

### 1.1 주요 기능

| 기능 | 설명 |
|------|------|
| 프로젝트 관리 | 프로젝트 생성·선택·설정 GUI |
| 팀 부팅 | AI 에이전트 팀 자동 시작 + 7단계 진행 시각화 |
| 팀 워크스페이스 | PM 채팅 + 6개 팀원 패널 동시 모니터링 |
| 산출물 뷰어 | 마크다운 산출물 열람·편집·버전 이력 |
| 레드마인 연동 | 이슈 조회·생성·상태 관리 |
| 내장 브라우저 | 앱 내 검색·분析·벤치마킹 |
| 설정 관리 | API 키 · 프로젝트 설정 통합 관리 |

### 1.2 시스템 요구사항

| 항목 | macOS | Windows |
|------|-------|---------|
| 최소 OS | macOS Ventura (13.0) | Windows 10 (1903) |
| 아키텍처 | Intel x64 / Apple Silicon | x64 |
| RAM | 4GB 이상 | 4GB 이상 |
| 디스크 | 200MB 이상 | 200MB 이상 |
| 인터넷 | AI API 호출 필요 | AI API 호출 필요 |

## 2. 설치

### 2.1 macOS 설치

1. 배포 페이지에서 `AgiTeamBuilder_vX.X.X_aarch64.dmg` 또는 `x64.dmg` 다운로드
2. DMG 파일 더블클릭 → 마운트
3. `AgiTeamBuilder Desktop.app`을 `/Applications` 폴더로 드래그
4. 최초 실행 시 Gatekeeper 경고 → **시스템 설정 → 개인 정보 보호 및 보안 → 열기** 허용
5. 앱 실행 확인

### 2.2 Windows 설치

1. 배포 페이지에서 `AgiTeamBuilder_vX.X.X_x64-setup.exe` 다운로드
2. 설치 파일 실행 → **사용자 계정 컨트롤(UAC)** 허용
3. 설치 마법사 안내에 따라 진행 (기본 경로: `C:\Program Files\AgiTeamBuilder Desktop\`)
4. 바탕화면 또는 시작 메뉴에서 앱 실행
5. **SmartScreen 경고** 발생 시 → "추가 정보" → "실행" 클릭

## 3. 최초 설정

### 3.1 API 키 등록

앱 최초 실행 후 **설정(Screen-07)** 화면이 열립니다.

1. 사용할 AI 공급자 선택: Claude / OpenAI / Gemini
2. 해당 API 키를 입력 필드에 붙여넣기
3. **[저장]** 버튼 클릭
   - 키는 OS 키체인(macOS) 또는 자격증명 관리자(Windows)에 암호화 저장
   - 평문으로 파일에 기록되지 않음

### 3.2 레드마인 연동 설정 (선택)

1. 설정 화면의 **레드마인** 탭 선택
2. 레드마인 서버 URL 입력 (예: `http://your-server:8080/`)
3. API 키 입력 (레드마인 내 내 계정 → API 접근 키)
4. **[연결 테스트]** → 초록색 성공 표시 확인
5. **[저장]**

---

## 4. 프로젝트 생성 및 선택 (Screen-01, Screen-02)

### 4.1 새 프로젝트 만들기

1. 런처 화면(Screen-01)에서 **[+ 새 프로젝트]** 클릭
2. 프로젝트 이름 입력 (예: `my-project`)
3. 저장 위치 선택
4. **[만들기]** → 기본 폴더 구조 자동 생성

### 4.2 기존 프로젝트 열기

1. 런처 화면에서 최근 프로젝트 목록 중 선택, 또는
2. **[폴더 열기]** → 기존 `agiteam.json`이 있는 폴더 선택

### 4.3 팀 구성 설정 (Screen-02)

1. 프로젝트 설정 화면에서 팀원 카드 목록 확인
2. 각 팀원 카드: AI 공급자(Claude/OpenAI/Gemini), 모델명, 페르소나 파일 경로 확인·수정
3. 팀원 추가: **[+ 팀원 추가]** → 역할 선택 → 설정 입력
4. 팀원 삭제: 카드 우측 **[✕]** 클릭
5. **[저장]** 후 **[팀 부팅 시작]** 클릭

---

## 5. 팀 부팅 (Screen-03)

팀 부팅은 7단계로 진행됩니다. 각 단계는 순서대로 자동 완료됩니다.

| 단계 | 내용 | 예상 시간 |
|:----:|------|:--------:|
| 1 | 환경 검증 (API 연결·키 확인) | ~2초 |
| 2 | 페르소나 번들 생성 | ~1초 |
| 3 | 팀원 순차 시작 | ~5~10초 |
| 4 | READY 신호 수신 대기 | ~10~30초 |
| 5 | PM 에이전트 시작 | ~5초 |
| 6 | 시작 파일 로드 | ~3초 |
| 7 | 워크스페이스 진입 | 즉시 |

- **🔴 오류 발생 시**: 오류 메시지 확인 후 **[재시도]** 클릭
- **API 연결 실패**: 설정 화면에서 API 키 재확인

---

## 6. 팀 워크스페이스 (Screen-04)

### 6.1 화면 구성

```
┌─────────────────────────────────────────────────────┐
│  [프로젝트명]  [산출물] [레드마인] [브라우저] [팀 종료] │
├───────────────────┬─────────────────────────────────┤
│                   │  Arch 🟢  DevOps 🟢               │
│  PM 채팅 패널     │  BE 🟡    FE 🟢                   │
│  (스트리밍 출력)  │  Designer 🟢  QA 🟢               │
│  [입력창]  [전송] │                                   │
└───────────────────┴─────────────────────────────────┘
```

### 6.2 PM 채팅 패널 (좌측)

- PM 에이전트의 출력이 실시간 스트리밍으로 표시됩니다
- 입력창에 지시 내용 입력 후 **[전송]** 또는 **Enter** 로 제출
- 산출물 파일 링크 클릭 → 산출물 뷰어로 이동

### 6.3 팀원 패널 (우측 2열×3행)

각 팀원 패널에 상태 뱃지가 표시됩니다:

| 뱃지 | 의미 |
|:----:|------|
| 🟢 READY | 지시 대기 중 |
| 🟡 BUSY | 작업 진행 중 |
| 🔴 ERROR | 오류 발생 |
| 🔵 APPROVAL | 승인 입력 대기 중 |

- **🔵 APPROVAL 표시 시**: 패널 확인 후 **[승인]** 또는 **[거부]** 클릭
- 패널 확대: 타이틀 바 우측 **[⛶]** 클릭

---

## 7. 산출물 뷰어/편집기 (Screen-05)

### 7.1 산출물 열기

- 워크스페이스 헤더 **[산출물]** 클릭 → 파일 브라우저
- PM 채팅 출력의 파일 링크 클릭

### 7.2 산출물 편집

1. 우측 상단 **[편집]** 버튼 클릭 → 편집 모드 진입
2. 마크다운 직접 수정
3. **[저장]**: 자동으로 현재 파일을 `_archive/<이름>_YYYYMMDDhhmmss.md`로 백업 후 `.latest.md` 갱신

### 7.3 버전 이력 보기

- **[이력]** 탭 클릭 → `_archive/` 폴더의 과거 버전 목록
- 과거 버전 클릭 → 현재 버전과 차이점(diff) 표시

---

## 8. 레드마인 연동 (Screen-06)

1. 헤더 **[레드마인]** 클릭
2. 이슈 목록 자동 로드 (프로젝트의 열린 이슈)
3. **[+ 새 이슈]**: 제목, 설명, 담당자, 트래커 선택 후 **[등록]**
4. 이슈 클릭 → 상세보기 → 상태 변경 / 코멘트 추가

---

## 9. 내장 브라우저

1. 헤더 **[브라우저]** 클릭
2. 상단 주소창에 URL 입력 또는 검색어 입력
3. 일반 웹 브라우저처럼 탐색 가능
4. 페이지 내용을 PM 채팅에 붙여넣기하여 AI 분析에 활용

## 10. 자주 묻는 질문 (FAQ)

**Q. API 키가 저장되지 않는다.**  
A. macOS: 키체인 접근 권한을 시스템 환경설정에서 확인. Windows: 자격증명 관리자 쓰기 권한 확인.

**Q. 팀 부팅 중 4단계(READY 대기)에서 멈춘다.**  
A. AI API 응답 지연일 수 있습니다. 30초 대기 후 타임아웃되면 자동 오류 표시. [재시도] 클릭.

**Q. 팀원 패널이 계속 🔴 ERROR 상태다.**  
A. 해당 팀원의 AI 공급자 API 키를 확인하거나, 다른 모델로 교체 후 재부팅.

**Q. 산출물 파일이 보이지 않는다.**  
A. 프로젝트 폴더 내 `documents/` 하위에 `.latest.md` 파일이 있는지 확인.

**Q. Windows에서 앱이 열리지 않는다.**  
A. WebView2 런타임이 설치되어 있어야 합니다. Microsoft WebView2 런타임을 별도 설치 후 재시도.
