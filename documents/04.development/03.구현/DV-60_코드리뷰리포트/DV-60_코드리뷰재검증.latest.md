### 3.2 DV60-003 Redmine IPC

| command | Rust command | lib.rs 등록 | 프론트 호출 | 판정 |
|---------|--------------|-------------|-------------|:---:|

| `redmine_create_issue` | 확인 | 확인 | `src/ipc/redmine.ts` wrapper 경유 호출 | PASS |
| `redmine_update_issue` | 확인 | 확인 | `src/ipc/redmine.ts` wrapper 경유 호출 | PASS |

최종 확인:

- `system/tos/src/ipc/redmine.ts` 신규 wrapper 존재 확인.
- `stores/redmine.ts`와 `RedmineView.vue`에서 직접 `invoke()` 사용 없음 확인.
- `RedmineView.vue`는 mock 직접 로드 대신 `redmineStore.fetchIssues/createIssueApi/updateIssueApi` 액션을 호출한다.
- DS-60 v0.3 §3.7은 `redmine_get_issue`만 상세 명세하고 list/create/update 상세 명세가 부족하므로 문서 보강은 권장 개선으로 유지한다.

### 3.3 DV60-004 Credential 저장

| 기준 | 근거 |
|------|------|
| 평문 상태 보관 제거 | `settings.ts`가 `redmineApiKeyStored: Record<string, boolean>`만 보관 |
| OS vault 저장 | `saveCredential('redmine', 'api_key', key)` 호출 |
| Rust provider kind | `AiProviderKind::Redmine` 추가 확인 |

주의:

- 역할별 입력 UI가 있지만 실제 저장 account는 공통 `api_key` 하나다. 역할별 API key가 요구되면 account 전략을 `api_key_<role>`로 재정의해야 한다.

### 3.4 DV60-005 설정 저장

| 기준 | 근거 |
|------|------|
| 프론트 IPC wrapper | `src/ipc/workspace.ts`에 `saveWorkspaceConfig()` 추가 |
| View 연결 | `ProjectSettingsView.vue` 저장 시 wrapper 호출 |
| Rust command | `commands/workspace.rs`에 `save_workspace_config` 구현 |
| lib 등록 | `lib.rs` invoke handler 등록 |

주의:

- `workspaceId`가 없는 신규 프로젝트 모드에서는 아직 메모리 저장만 수행한다. 기존 workspace 편집 결함은 해소됐지만 신규 프로젝트 파일 생성 완료 검증은 별도 시나리오가 필요하다.

### 3.5 DV60-006 문서 저장 DTO

| 기준 | 근거 |
|------|------|
| Rust 타입 | `path`, `archive_path: Option<String>`, `version_hint` |
| TS 타입 | `path`, `archive_path?`, `version_hint` |
| View 사용 | `afterSave(result.archive_path ?? null)` |

잔여 문서 정합성:

- DS-60 v0.3의 `DocumentWriteResult`에는 `last_updated: string`이 있으나 Rust/TypeScript 구현에는 없다. Rust↔TypeScript 정합성은 PASS이나 DS-60 v0.3↔구현 정합성은 보완 필요.
