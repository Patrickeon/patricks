# TS-15 시험시나리오 - AgiTeamBuilder Desktop

> **목적**: TS-05 시험계획서에 따라 AgiTeamBuilder Desktop의 주요 화면, 크로스플랫폼 동작, 성능·보안 비기능 요구사항을 정적 재현과 동적 재현 방식으로 검증할 수 있는 시나리오를 정의한다.

## 1. 공통 기준

### 1.1 시나리오 판정 체계

| 판정 | 기준 |
|------|------|
| PASS | 정적 재현과 동적 재현이 모두 성공하고 기대 결과와 실제 결과가 일치 |
| FAIL | 요구사항 또는 화면설계 기준과 실제 결과가 불일치 |
| BLOCKED | 테스트 환경, 빌드, 계정, API 키, 외부 서비스 접근권한 부재로 실행 불가 |
| N/A | 현 단계 구현 범위 밖이거나 PM이 제외 승인한 항목 |

### 1.2 정적 재현 / 동적 재현 기록 원칙

| 단계 | 기록 내용 |
|------|-----------|
| 정적 재현 | AN-40 요구사항 ID, DS-50 화면/컴포넌트/이벤트 기준, 구현 코드 또는 설계 대조 결과 |
| 동적 재현 | macOS/Windows 또는 테스트 harness에서 실행한 단계, 로그, screenshot, Playwright trace 등 관찰 증거 |

### 1.3 공통 테스트 데이터

| 데이터 | 값 |
|--------|----|
| 테스트 프로젝트명 | `tos-qa-fixture` |
| 테스트 workspace | `/tmp/agiteam-tos-qa` 또는 OS별 임시 디렉토리 |
| 테스트 agiteam.json | 정상 6역할 구성 + 오류 케이스 fixture |
