# 04.테스트/

> 구현 결과를 **검증**하는 테스트 산출물(TS)을 둔다.

| 하위 | 무엇 | 담당(R) |
|---|---|:---:|
| `TS-05_시험계획서/` | 테스트 범위·방법·일정·환경·역할 계획 | QA |
| `TS-10_단위테스트결과서/` | 기능 단위 테스트 결과 | BE |
| `TS-15_시험시나리오/` | 재현 가능한 테스트 시나리오 | QA |
| `TS-16_테스트데이터/` | 시드·시나리오 템플릿·재현용 데이터셋 | QA |
| `TS-20_통합테스트결과서/` | 모듈·API·화면 통합 검증 결과 | QA |
| `TS-25_성능시험결과서/` | 응답시간·처리량·부하 등 비기능 검증 | QA |
| `TS-30_결함리스트/` | QA가 식별한 결함 목록 | QA |
| `TS-40_수정코드_INDEX/` | 결함 수정 코드 위치 참조 | BE/FE* |
| `TS-50_수정내역서/` | 결함별 수정 내용·조치 결과 | BE/FE* |
| `TS-60_릴리스승인서/` | 배포 가능 여부 승인 문서 | QA |
| `TS-70_보안취약성점검결과서/` | 보안 취약점 스캔·진단·조치 결과 | DevOps |

> `*` TS-40·TS-50: 결함을 일으킨 산출물의 R 담당자가 R이 된다 (BE 코드 결함이면 BE, FE 코드 결함이면 FE).
