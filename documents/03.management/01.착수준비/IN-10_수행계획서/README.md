# IN-10 수행계획서 작성요령

> 이 문서는 **작성자(PM)** 와 **QA(감리)** 가 읽는 가이드입니다.

## 1. 목적
프로젝트 전체를 지휘하는 최상위 계획 문서. 범위·일정·조직·품질·위험·보안 전 영역 기준선.

## 2. 담당 / 참조
- **주담당**: PM
- **참조·검토**: 유저 (승인), QA (감리)

## 3. 선행 산출물 (Input)
- `01.proposal/` 진입 자료 (Forward) 또는 `02.reverse/` (Reverse)
- 유저 요구사항 인터뷰

## 4. 후속 산출물 (Output)
- `IN-20_WBS` (일정·태스크 분해)
- `IN-30_착수보고서`
- `산출물리스트.md` (하네스)
- 전체 DD 산출물의 기준선

## 5. 필수 섹션
1. 사업 개요 (사업명·기간·목적·유형)
2. 사업의 범위 (포함/제외/전제조건)
3. 사업 추진 체계 (팀·RACI·커뮤니케이션)
4. 문서 및 산출물 관리 (하네스)
5. 일정관리·품질관리·위험관리·보안관리
6. 페르소나·원형 관리 (AgiTeamBuilder 고유)
7. 변경 관리

## 6. 작성 요령
- **이 문서는 불가침**: 작성 후 유저 승인 없이 변경 금지
- 각 장은 "지시가 가능한 수준"까지 구체. 모호한 표현(적절히·가능한 한) 지양
- 하네스(산출물리스트)와 필드 일관성 유지

## 7. 작성자 체크리스트
- [ ] 사업 유형(Forward/Reverse/Internal) 명시
- [ ] Phase별 기간·범위 구체화
- [ ] 팀 7역할 RACI 완성
- [ ] 필수 설계 산출물 3종(API·ERD·테이블) 정합성 규칙 포함
- [ ] 레드마인·cmux 프로토콜 명시
- [ ] AI 에이전트 팀 고유 위험 섹션 포함
- [ ] 붙임(산출물리스트·WBS) 링크 유효

## 8. QA 감리 체크리스트
- [ ] 10장 구조 완비, 누락 장 없음
- [ ] 개정이력 승격 기록 확인
- [ ] 산출물리스트.md와 팀·담당역할 일치
- [ ] 수행계획서 내용과 실제 운영(cmux·레드마인) 괴리 없음

## 9. 자주 하는 실수
- 수행계획서와 산출물리스트 담당자 불일치
- Phase 범위가 너무 추상적이라 태스크 분해 불가
