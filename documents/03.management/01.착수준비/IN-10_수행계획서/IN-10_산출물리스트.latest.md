## 개요

본 리스트는 **두 트랙**으로 구성된다.

| 트랙 | 식별자 | 건수 | 설명 |
|------|:---:|:---:|------|
| **매트릭스 산출물** | AN / DS / DV / TS / IM / OP | **38** | RACI매트릭스 §4 기반 — 분석·설계·구현·테스트·이행·운영 6단계 산출물 흐름 |
| **메타 레이어** (관리산출물) | IN / MG | **12** | 프로젝트 관리·운영 통제 (RACI매트릭스 외부, 별도 트랙) |
| 진입자료 (Forward 전용) | PD | 4+ | RFP·제안서·계약·변경요청 |
| 진입자료 (Reverse 전용) | RD | 5+ | 인수·현황·자산·이관·운영인수 |
| **본체 합계** (매트릭스 + 메타) | - | **50** | |

## 5. 사용법

### 5.1 담당역할 → 실제 담당자 매핑

본 리스트의 **R 컬럼**은 7역할 원형(PM·Design·Arch·DevOps·BE·FE·QA) + 오케스트레이터 중 하나. 실제 프로젝트에서는 자식 인스턴스 담당자(예: 박피엠·김아키)가 해당 역할을 수행하되, 본 리스트는 **역할명을 유지**한다. (역할 ↔ 실제 담당자 매핑은 자식 `brain/Shared/team.md`에서 관리)

### 5.2 상태 업데이트 규칙

```
담당역할이 작업 착수     → 미작성 → 작성중
담당역할이 초안 완료     → 작성중 → 검토중 (QA 검토 대기)
QA 검토 통과             → 검토중 → 완료
상시 산출물은 생성 이후    → 갱신중 (계속 유지, 완료로 가지 않음)
```

### 5.3 제출일 기록

| 타입 | 제출일에 기록하는 값 |
|------|------------------|
| 일회성 | 최종 승인 날짜 |
| 상시 | 최근 갱신 날짜 |

### 5.4 Phase 확장

Phase가 여러 개인 프로젝트(Phase 1·2·3…)는 다음 중 택일:

- **방식 A**: 본 리스트 1개를 유지하고 비고 칸에 Phase 표시
- **방식 B**: Phase별로 산출물리스트를 복제 (`산출물리스트_Phase1.md` 등)

선택 기준은 각 프로젝트 자체 정책으로 결정.

### 5.5 파일명 규칙

각 산출물 폴더 내부 파일명 표준 (현행본은 `.latest` 고정 — 상세는 RACI매트릭스 §3.2~§3.4):

```
{ID}_{산출물명}.latest.{ext}

예:
  IN-10_수행계획서.latest.md
  AN-20_요구사항정의서.latest.md
  DS-20_아키텍처설계서.latest.md
```

**규칙**:
- **파일명에 날짜·버전 금지**: 현행본은 항상 `.latest` 고정 → 링크가 깨지지 않는다
- **버전·날짜는 frontmatter**: `version`·`last_updated` (RACI §3.3)
- **백업**: 과거본 보존 시 `{ID}_{산출물명}.{YYYYMMDDHHMMSS}.{ext}` (RACI §3.4, 링크 대상 아님)
- **프로젝트명 제외**: 상위 경로에서 구분됨
- **확장자**: `.md` 기본. 바이너리(이미지·엑셀 등)는 해당 확장자 유지

### 5.6 식별자(ID) 규칙

| Prefix | 의미 | 적용 |
|:---:|------|------|
| AN | Analysis | 분석 산출물 (`04.development/01.분석/`) |
| DS | Design | 설계 산출물 (`04.development/02.설계/`) |

| IM | Implementation | 이행 산출물 (`04.development/05.이행/`) |
| OP | Operation | 운영 산출물 (`05.operation/`) |
| IN | Initiation | 착수준비 관리산출물 (`03.management/01.착수준비/`) |
| MG | Management | 상시관리 관리산출물 (`03.management/02.상시관리/`) |

- **AN/DS/DV/TS/IM/OP**: `[단계]-[10단위]` (사이 삽입 필요 시 5단위 — 예: TS-05·IM-05·OP-05)

## 7. 주의사항

- 본 리스트는 **하네스**다. 리스트에 없는 산출물은 원칙상 생산하지 않는다.
- 새 산출물 추가 시 PM이 본 리스트에 항목을 먼저 등록한 후 담당역할에 지시한다.
- 상시 산출물은 프로젝트 내내 살아있는 문서이므로 `완료` 상태로 전환하지 않는다.
- **본 리스트는 RACI매트릭스의 부속 문서**. 매트릭스 갱신 시 본 리스트도 함께 갱신해야 한다.
- 폴더 구조(`04.development/01.분석/`·`02.설계/`·`03.구현/`·`04.테스트/`·`05.이행/`·`05.operation/`)는 `00.standard/폴더구조.latest.md`에 정의되어 있다. 폴더구조 표준 갱신 시 본 리스트도 정합화한다.
