# 프로젝트 WBS — Reverse 전용 (역공학)

> **적용 모드**: Reverse(기존 운영처 인수·역공학). 흐름이 반대 — `돌아가는 시스템(코드·DB·운영) → 설계·요구사항` 역방향. 코드가 설계를 낳는다(역추출).
> **기준 매트릭스**: `00.standard/RACI매트릭스` §4.4 (Reverse 전용 19행) + §7.5 (Reverse 모드 정의).
> **목적**: 운영 모드 진입을 위한 baseline 설계도를 역으로 복원. 1회성(프로젝트 초기 1 Phase). 종착 = 인수 후 baseline 충족 → 운영 모드 합류.
> **Forward/Internal 프로젝트는 별도 WBS** — `IN-20_WBS_Forward_Internal.latest.md` 사용.
> **실전 검증**: KEAPMS(KEA PMS 2차 사업 인수, Spring5·eGov3.7·MyBatis·MariaDB·Tomcat9·JDK17, 설계 산출물 0건 → 전량 역추출).

## 0. 작성·갱신 규칙 (필수 숙지)

| 항목 | 규칙 |
|------|------|
| 기준 문서 | IN-10 수행계획서(범위·일정) + **RACI매트릭스 §4.4**(Reverse 활동·산출물·R담당·인계자산) |
| 단계 골격 | **환경 → 분석(역도출) → 설계(복원) → 시각 → 시험 → 이행 → 게이트 → 운영합류** (Forward의 단계와 다름) |
| 흐름 방향 | **역방향** — 인계 자산(코드·dump.sql·돌아가는 시스템)이 입력, 설계도가 출력 |
| WBS ID | WBS 안에서만 쓰는 행 식별자. 복원 대상 산출물 코드를 빌려 표기 |
| 점진적 상세화 | 인수 전 baseline 복원 → 인수 게이트 → 인수 후 실운영 관찰로 정적추론 승격 |
| 담당(R) | RACI §4.4의 R 그대로. **분석 자료는 코드 읽는 BE·FE가 R**, 설계도 복원은 Architect, 시각은 Designer |
| 선행 | RACI §4.4 "선행"의 활동 |
| 갱신 시점 | ① 분석 완료 ② 설계 복원 완료 ③ 인수 게이트 통과 ④ 태스크 완료 시 상태 |
| 상태 범례 | `대기` / `진행` / `완료` / `보류` |
| 판정 라벨 | `식별`→`정적근거`→`정적재현`→`동적재현`→`운영확인` (RACI매트릭스 §7.5) |

> **인수 게이트**: 인수 전 baseline은 인수 **의사결정** 게이트(A=오케스트레이터). 인수 후 baseline은 실운영 관찰로 정적추론을 승격해야 운영 모드 정상 가동 (RACI매트릭스 §7.5).
