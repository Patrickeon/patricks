# IN-20 WBS 작성요령

> 이 문서는 **작성자(PM)** 와 **QA** 가 읽는 가이드입니다.

## 0. WBS 파일 — 사업 유형별 2종 (정식 채택)

본 폴더에는 사업 유형에 따라 **2개의 WBS 템플릿**이 있다. PM은 프로젝트 모드 진입 시 사업 유형에 맞는 파일을 **복사하여 정식 WBS로 사용·관리**한다.

| 사업 유형 | 사용 WBS 파일 | 기준 매트릭스 | 흐름 |
|---|---|---|---|
| **Forward · Internal** | ⭐ `IN-20_WBS_Forward_Internal.latest.md` | RACI §4.3 (42행) | 요구사항→설계→코드 (순방향, 신규 구축) |
| **Reverse** | `IN-20_WBS_Reverse.latest.md` | RACI §4.4 (19행) | 시스템→설계 (역방향, 역공학 복원) |

- **정식 WBS(프로젝트 모드 기본)** = `IN-20_WBS_Forward_Internal` — 신규 구축이 일반 케이스. PM이 이 파일을 복사·갱신하며 프로젝트를 리딩한다.
- **Reverse 프로젝트**일 때만 `IN-20_WBS_Reverse`를 사용. 운영 모드 진입용 baseline 역추출 중심.
- 두 파일 모두 **점진적 상세화(Rolling Wave)** — 착수 시 골격, 분석·설계 종료 시 PM이 상세 보강.

## 1. 목적

수행계획서를 태스크 단위로 분해. 일정·의존관계·담당자 매핑. 레드마인 이슈 생성의 원천.

## 2. 담당 / 참조

- **주담당**: PM
- **참조**: 전 팀원 (본인 작업 확인)

## 3. 선행 산출물 (Input)

- `IN-10_수행계획서` (범위·일정)
- `IN-10_산출물리스트.md` (산출물 단위로 태스크 생성)

## 4. 후속 산출물 (Output)

- 레드마인 이슈 (tracker_id=2 새기능, 하위일감으로 분배)
- `IN-30_착수보고서` (킥오프 자료)

## 5. 필수 섹션

1. Phase별 태스크 트리 (WBS 계층)
2. 태스크별 담당역할·예상 기간·선행 태스크
3. 마일스톤
4. 크리티컬 패스 (선택)

## 6. 작성 요령

- 태스크 단위는 **1~5일 범위**. 너무 크면 진척 추적 어려움
- 선행-후속 관계 명시 (의존 없는 태스크는 병렬 실행 가능)
- 담당역할은 7역할 중 택1. 2명 공동이면 주담당 지정
- WBS ID 부여 (예: Phase1-1.1.1)

## 7. 작성자 체크리스트

- [ ] 모든 산출물이 최소 하나의 태스크로 커버됨
- [ ] 담당역할 공란인 태스크 없음
- [ ] 선행 태스크 체인에 순환 참조 없음
- [ ] Phase 경계 태스크(착수/종료) 포함
- [ ] 레드마인 이슈 번호 매핑 (생성 후)

## 8. QA 감리 체크리스트

- [ ] WBS 태스크 ↔ 산출물리스트 1:N 매핑 확인
- [ ] 예상 기간 합이 전체 사업 기간과 일치
- [ ] 크리티컬 패스의 태스크에 여유 시간 0
- [ ] 레드마인 이슈 상태와 WBS 진척율 동기화

## 9. 자주 하는 실수

- 태스크를 너무 크게 (`"Phase1 개발"` 단일 태스크)
- 의존관계 누락 → 병렬 착수 실패

## 10. 파일명·버전

- Forward/Internal: `IN-20_WBS_Forward_Internal_YYYYMMDD_v0.1.md`
- Reverse: `IN-20_WBS_Reverse_YYYYMMDD_v0.1.md`
