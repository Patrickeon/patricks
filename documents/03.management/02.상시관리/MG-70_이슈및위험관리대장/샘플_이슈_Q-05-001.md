# Q-05-001 사용자 수정 시 음수 연차 입력으로 500 오류 발생

> 작성일: 2026-04-04
> 작성자: Codex-QA
> 심각도: 높음
> 상태: Open

---

## 1. 요약

`PATCH /api/users/{userId}` 호출 시 `annual_leave_total=-1`을 전달하면 명세상 `400 VALIDATION_ERROR`가 반환되어야 하나, 실제로는 `500 Internal Server Error`가 발생한다.

---

## 2. 재현 절차

1. 관리자 계정으로 로그인해 액세스 토큰을 발급받는다.
2. 테스트용 사용자를 생성한다.
3. 아래 요청을 호출한다.

```bash
curl -i -X PATCH http://localhost:11001/api/users/{userId} \
  -H "Authorization: Bearer <ADMIN_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"annual_leave_total":-1}'
```

---

## 3. 기대 결과

- HTTP `400`
- 에러 코드: `VALIDATION_ERROR`
- 음수 연차 입력 불가 메시지 반환

---

## 4. 실제 결과

- HTTP `500`
- 응답 본문:

```text
Internal Server Error
```

---

## 5. 영향도

- 잘못된 입력이 서버 예외로 전파되어 API 안정성을 저하시킨다.
- 프론트엔드는 사용자 입력 오류와 서버 장애를 구분할 수 없다.
- 운영 환경에서 동일 계열 입력이 반복되면 장애성 로그가 누적될 수 있다.

---

## 6. 추정 원인

- 요청 스키마 검증 또는 서비스 계층 검증에서 음수 연차를 사전에 차단하지 못하고 있다.
- DB 제약조건 또는 후속 처리 중 예외가 발생해 공통 예외 변환 없이 500으로 노출되는 것으로 보인다.

---

## 7. 수정 권장사항

- 요청 스키마에서 `annual_leave_total >= 0` 제약을 명시한다.
- 서비스 계층에서도 동일 검증을 수행한다.
- 검증 예외를 공통 에러 응답(`400 VALIDATION_ERROR`)으로 변환한다.
