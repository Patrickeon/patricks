# MG-70 이슈및위험관리대장 작성요령

> 이 문서는 **작성자(PM)** 와 **QA·전 팀원** 이 읽는 가이드입니다.
> **상시 산출물** — 이슈 발견 시마다 누적.

## 1. 목적
프로젝트 전체의 이슈·위험을 통합 추적. 레드마인과 연계.

## 2. 담당 / 참조
- **주담당**: PM
- **참조**: QA (결함 등록), 전 팀원 (발견 보고)

## 3. 선행 산출물 (Input)
- QA 검증·회귀 테스트 결과
- 레드마인 결함 티켓
- 유저 제보

## 4. 후속 산출물 (Output)
- 심각도 Critical/High는 `MG-10_중간보고서`에 즉시 반영

## 5. 필수 컬럼
1. ID (레드마인 이슈 번호 또는 IR-NNN)
2. 발견 일자
3. 발견자
4. 유형 (이슈·위험·결함)
5. 설명 (재현·기대·실제)
6. 심각도 (Critical/High/Medium/Low)
7. 영향 범위
8. 대응 방안
9. 담당자
10. 상태 (식별·분석·대응·종결)
11. 종결 일자

## 6. 작성 요령
- **결함**은 재현 절차·기대 결과·실제 결과 필수
- **위험**은 발생 확률·영향도 별도 표기
- 종결 시 "어떻게 해결했는지" 1~2줄 기록 (미래 참조)

## 7. 작성자(PM) 체크리스트
- [ ] 재현 절차 구체적 (코드 단위)
- [ ] 심각도 판단 근거 기재
- [ ] 담당자 지정

## 8. QA 감리 체크리스트
- [ ] 레드마인 상태와 대장 상태 동기화
- [ ] Critical·High 적체 없음
- [ ] 종결 건이 실제 수정 반영됐는지

## 9. 자주 하는 실수
- 레드마인에만 기록하고 대장 미갱신
- 유저 제보 결함을 내부 대장에만 기록 (유저에게 피드백 누락)

## 10. 파일명·버전
상시 단일: `MG-70_이슈및위험관리대장_YYYYMMDD_v0.1.md`

## 11. 샘플
