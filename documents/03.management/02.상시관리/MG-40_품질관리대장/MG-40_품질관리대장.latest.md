## 결함 상세

### DV60-001 [Critical] boot_team PM 부팅 누락

- **현상**: 프론트 `BootView.vue`는 PM을 포함한 7개 에이전트 READY를 대기하나, Rust `boot_team` 커맨드가 `agiteam.json`의 roles 배열(팀원 6명)만 순회해 PM을 별도 부팅하지 않음
- **영향**: 부팅 7단계 중 PM READY 신호가 오지 않아 워크스페이스 진입 무한 대기
- **권고**: `boot_team`에서 PM 역할도 별도 AgentSession 생성 + `pm_ready` 이벤트 emit 또는 PM 전용 `boot_pm` 커맨드 추가

### DV60-003 [High] Redmine 실 API 미연동

- **현상**: `RedmineView.vue`는 하드코딩 mock 데이터를 표시하고, `useRedmineStore`의 fetch/create/update 액션이 실제 `invoke('redmine_*')` 호출 없이 로컬 상태만 조작
- **영향**: 레드마인 이슈 조회·생성·상태 변경 전혀 동작하지 않음
- **권고**: DS-40 §레드마인 API 기반으로 Rust `redmine_client.rs` 구현 + Tauri invoke 연동

### DV60-004 [High] Redmine API 키 평문 저장

- **현상**: `useRedmineStore.setApiKey(key)`가 Pinia 상태에 문자열로 저장, `localStorage` 또는 파일에 평문 기록 가능
- **영향**: API 키 노출 (NFR-0301 보안 요구사항 위반)
- **권고**: `invoke('save_credential', { provider: 'redmine', ... })` → OS 키체인 저장으로 교체

### DV60-005 [High] agiteam.json GUI 저장 미동작

- **현상**: `ProjectSettingsView.vue`의 [저장] 버튼이 `useProjectStore.updateConfig()`만 호출하고 `invoke('save_workspace_config')` 미호출 → 앱 재시작 시 변경사항 유실
- **권고**: 저장 버튼 핸들러에 `invoke('save_workspace_config', { config })` 추가

### DV60-006 [Medium] 산출물 저장 DTO 불일치

- **현상**: Rust `write_latest_document` 반환값 `{ success: bool, path: String }`이나 TypeScript `writeLatestDocument()` 반환 타입이 `void`로 선언
- **영향**: 저장 성공/실패 피드백 불가, 오류 처리 누락
- **권고**: TypeScript IPC 래퍼 반환 타입 `Promise<{ success: boolean; path: string }>` 수정
