# MG-40 품질관리대장 작성요령

> 이 문서는 **작성자(QA)** 와 **PM** 이 읽는 가이드입니다.
> **상시 산출물** — 감리 발견 시마다 누적.

## 1. 목적
산출물 품질 검토 이력 누적. 코드-설계 불일치, 산출물 간 정합성 결함을 추적·종결.

## 2. 담당 / 참조
- **주담당**: QA
- **참조**: PM (심각도 판단), Architect (설계 문서 수정 지시 수신)

## 3. 선행 산출물 (Input)
- 모든 설계 산출물 (감리 대상)
- 코드 (감리 대상)

## 4. 후속 산출물 (Output)
- 결함 → 레드마인 이슈 (tracker_id=1)
- 정합성 이슈 → Architect에게 설계 수정 지시

## 5. 필수 컬럼 (표 형식)
1. ID (QI-NNN)
2. 검토 일자
3. 검토 대상 (산출물 ID / 코드 경로)
4. 불일치 항목
5. 심각도 (Critical/High/Medium/Low)
6. 조치 방안
7. 담당자
8. 상태 (열림/진행/해결/종결)
9. 종결 일자

## 6. 작성 요령
- 새 행 **추가**, 기존 행 **수정 금지** (감사 이력 보존)
- 상태 변경은 별도 컬럼이 아닌 같은 행 갱신 + 종결 일자 기록
- 심각도 판단은 "7.2 위험 평가 기준" 참조

## 7. 작성자(QA) 체크리스트
- [ ] 새 ID는 연번 중복 없음
- [ ] 불일치 항목을 재현 가능한 형태로 기재 (파일:라인)
- [ ] 담당자 지정 (PM이 역할 할당)

## 8. PM 감리 체크리스트
- [ ] 열림 건 주간 점검 (MG-30에 보고)
- [ ] 종결 건이 실제 산출물에 반영됐는지 확인
- [ ] Critical·High 심각도 적체 없음

## 9. 자주 하는 실수
- 구두로만 해결하고 대장에 종결 미기재 (이력 소실)
- 심각도 과소평가

## 10. 파일명·버전
상시 단일 파일: `MG-40_품질관리대장_YYYYMMDD_v0.1.md` — 마이너 수정은 내부 개정이력 누적 권장.

## 11. 샘플
`샘플_품질관리대장.md` (Supports). 실제 감리 이력 형식 참고.
