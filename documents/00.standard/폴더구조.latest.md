# AgiTeamBuilder 폴더구조 표준

> AgiTeamBuilder가 생성하는 모든 자식 프로젝트의 **공통 폴더 구조 기준 문서**.
> RACI매트릭스의 산출물 흐름이 어떤 폴더에 배치되는지를 정의한다.

---

## 1. 적용 범위

- **신규 생성 자식 프로젝트** — 본 표준 채택 시점 이후 `make.sh`로 생성되는 모든 자식

---

## 3. 폴더별 상세

### 3.1 `00.standard/`

RACI매트릭스 + 본 폴더구조 표준 + 산출물리스트. **모든 모드 공통**. 자식 프로젝트는 본 폴더를 수정하지 않고 원형 그대로 상속.

### 3.2 `01.proposal/` (Forward 전용)

RFP·제안서·계약서·변경요청. Forward 모드 진입자료. 다른 모드에서는 빈 폴더 또는 미생성.

### 3.3 `02.reverse/` (Reverse 전용)

인수자료·현황분석·자산인벤토리·이관계획·운영인수. Reverse 모드 진입자료. 다른 모드에서는 빈 폴더 또는 미생성.

### 3.4 `03.management/` (관리 산출물, 모든 모드)

**2그룹 구조**:

- **`01.착수준비/`** — 착수 시점 일회성 산출물 (IN-10 수행계획서·IN-20 WBS·IN-30 착수보고서 + IN-40~60 품질보증·위험관리·형상관리 계획서). 착수준비(IN) 단계 = RACI §3.1 단계 코드.
- **`02.상시관리/`** — 프로젝트 내내 상시 갱신 (MG-10 중간·MG-20 완료·MG-30 주간 보고서 + MG-40~90 품질·보안·변경·이슈위험·회의록·ActionItem 대장).

**Operation에서도 필수** — 운영 중 장애·변경·회의·이슈·액션은 02.상시관리에 누적.

### 3.5 `04.development/` (매트릭스 산출물)

AN·DS·DV·TS·IM 5단계. 자식 폴더는 흐름 순서 알파벳 정렬을 위해 `01.분석/`·`02.설계/`·`03.구현/`·`04.테스트/`·`05.이행/` prefix 적용 (DV 자식은 부모 "개발"과 어감 중복 회피 위해 "구현"). Forward·Internal은 풀세트, Reverse는 중심(산출물별), Operation은 참조용.

### 3.6 `05.operation/` (운영 모드 + 앞 모드 종료 후)

OP 4건 (기술적용결과표·모니터링·장애 리포트·장애 조치). Operation 모드의 본 영역.

### 3.7 `_archive/` (선택)

폐기·이관 흔적 보존. 옛 DD 폴더 이관 시 `legacy-DD_yyyymmdd/`에 보관 + 매핑표 README 필수.

## 4. 운영 규칙

### 4.1 모드 정의 = RACI매트릭스 §7로 통합 (단일 원천)

- 옛 `mode/` 폴더(forward·internal·reverse·operation.md)는 **RACI매트릭스 §7 「모드별 운영 정의」로 통합**되었다 (SSOT — 모드 정의의 단일 원천).
- 모드 공통 규칙(폴더·SSOT·결함ID·정합매핑)은 RACI §7.1, 모드별 고유(정의·흐름·판정·종료·평가)는 RACI §7.3~§7.6.
- 산출물은 **ID 기반 참조**(예: `AN-20`·`DS-40`) — 산출물명 변경 안전성.

### 4.2 RACI 변경 시 정합 유지

- RACI 산출물 ID 추가·삭제·명칭 변경 시 영향받는 산출물(WBS·폴더 등) 동시 갱신.
- QA 정기 감리: "RACI에 있는데 폴더/WBS에 없는 항목" / "폴더/WBS에 있는데 RACI에 없는 항목".

### 4.3 코드 산출물 인덱스

- `system/` 아래 실코드 산출물(DV-20·30·40·TS-40·DV-50 일부)은 `04.development/03.구현/<ID>_<이름>_INDEX/` 및 `04.development/04.테스트/<ID>_<이름>_INDEX/` 폴더에 위치 참조 README·링크 산출물 필수
- 미명시 시 RACI 산출물 목록상 존재하지만 문서 트리에서 누락된 것처럼 보임

### 4.4 폴더 신설·삭제 절차

- 신설: 새 산출물 추가 시 본 표준 갱신 → 산출물리스트 갱신 → 폴더 신설
- 삭제: 폴더는 삭제하지 않고 `_archive/`로 이관
- 이관 시 매핑표 README 동봉 (구 위치 ↔ 신 위치)

## 5. 옛 폴더 처리 정책

### 5.1 옛 DD010~DD250 25개 폴더 (적용 중)

기존 AgiTeamBuilder/AgiTeam/documents/04.development/ 아래 DD010~DD250 폴더는 다음 절차로 처리:

1. **`_archive/legacy-DD_yyyymmdd/`로 일괄 이관** (`git mv`)
2. `_archive/legacy-DD_yyyymmdd/README.md` 신설 — 구 DD ↔ 신규 산출물 코드 매핑표 포함 (RACI매트릭스 §7 활용)
3. 유효 산출물만 PM 승인하에 새 구조로 선별 이관
4. 이관 상태 기록 (이관 완료 / 폐기 / 보류) — archive README 갱신

### 5.2 일반 원칙

- **즉시 삭제 금지** — 감사 추적성·링크 보존
- **이관 전 매핑표 필수** — 이중 관리 방지
- **PM 승인 절차** — 산출물별 이관·폐기·보류 결정
