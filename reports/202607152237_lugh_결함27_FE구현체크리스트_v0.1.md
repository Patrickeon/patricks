## FE 구현 체크리스트

- **내용**: `useProjectStatus()` — `none`(`config===null`) / `configured`(config 有·미부팅) / `active`(`workspaceStore.isActive`) 3상태 파생. 신규 영속 상태 불필요(기존 스토어 교차 파생).
- **대상 파일**: `src/composables/useProjectStatus.ts` (신규)
- **BE 작업**: 없음

### 2. 라우터 가드 완화

- **내용**: 팀 종료 확인 → `stop_role`(세션 종료) → `projectStore.reset()` + `workspaceStore.deactivate()` + `roleStore` 초기화 → (새 프로젝트)`load`→`/boot`→`/workspace` / (닫기)`/launcher`.

- **대상 파일**: `src/views/LauncherView.vue` · `src/views/WorkspaceView.vue` (공통)
- **BE 작업**: 없음
- **근거**: DS-10 §8.2 기능 가용성 매트릭스

## 참고

- **8·9번 주의**: `roleStore`에 일괄 초기화 액션이 없으면 신설 필요(전환 시 이전 프로젝트 역할 상태 잔존 방지). 나머지 스토어 액션(`project.reset` / `workspace.deactivate`)은 기존 존재 확인됨.
- **상태 전이**: `none →(생성/열기)→ configured →(부팅)→ active`, 역방향 `active →(닫기/전환)→ none`.
- **화면 방식**: `none`의 화면 실체는 `/launcher`(홈). 워크스페이스(`/workspace`)는 `active`에서만 진입(빈 워크스페이스 렌더 방식 미채택).
