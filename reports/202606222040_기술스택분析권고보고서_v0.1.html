Electron + TypeScript + React

내장 브라우저 패널(WebContentsView)이 핵심 요구사항인 경우 Electron이 완전한 Chromium 환경을 제공합니다. 단, 브라우저 패널이 "웹사이트 열 수 있으면 됨" 수준이라면 Tauri + Rust도 동등하게 유효한 선택입니다. 최종 결정은 아래 사실을 보고 오케스트레이터가 합니다.

참고 — .pkg vs .dmg: macOS 사용자가 앱을 설치할 때 99%는 .dmg (드래그 인스톨)를 사용합니다. .pkg는 기업 배포·시스템 수준 설치(LaunchDaemon 등)에 쓰입니다. 소비자 배포라면 .dmg만으로 충분합니다. 두 프레임워크 모두 단일 소스 코드베이스에서 Windows + macOS 설치 파일을 생성합니다. 단, 플랫폼별로 빌드 머신이 필요합니다 (Windows .exe는 Windows에서, .dmg는 Mac에서 빌드 — 또는 GitHub Actions CI로 자동화).
Q2 — Rust 로 해도 되지 않나? Tauri 는 뭔가?
Tauri = "Rust 백엔드 + 웹 프론트엔드" 프레임워크
Tauri는 Rust로 앱의 핵심 로직(파일 시스템·IPC·프로세스·자격증명·AI API 호출)을 작성하고, UI는 기존처럼 TypeScript + React(또는 Vue·Svelte)로 작성하는 구조입니다. Rust 코드와 JavaScript 코드가 하나의 앱 안에서 공존합니다.

현재 시스템(agiteam.sh)의 모든 bash 로직 — config 파싱, 에이전트 부팅, IPC, 파일 읽기 — 이 전부 Rust로 재작성되고, 화면(PM 패널·채팅·대시보드)만 TypeScript/React가 담당합니다. Python은 두 경우 모두 전혀 필요 없습니다.
순수 Rust GUI(iced·egui·Slint)도 존재하지만, 이 프로젝트에서 요구하는 "웹 브라우저 패널 내장"은 구현이 어렵거나 불가능합니다. 따라서 Rust를 선택한다면 현실적 선택지는 Tauri입니다.
